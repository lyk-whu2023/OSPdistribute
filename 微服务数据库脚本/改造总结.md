# 微服务数据库改造总结

## 📋 改造背景

原有的数据库脚本采用单数据库设计（`shopping_platform_db`），所有表都在同一个数据库中，存在以下问题：

1. ❌ **不符合微服务架构**：所有服务共用一个数据库，无法独立部署和扩展
2. ❌ **跨服务外键依赖**：存在大量跨表外键约束，导致服务耦合
3. ❌ **无法独立扩展**：无法对单个服务进行水平扩展
4. ❌ **单点故障风险**：单个数据库故障影响所有服务

## ✅ 改造方案

### 1. 数据库拆分

将原来的单数据库拆分为 **9 个独立数据库**，每个服务使用独立数据库：

| 服务 | 数据库名 | 表数量 |
|------|---------|--------|
| 用户服务 | user_db | 2 |
| 商品服务 | product_db | 9 |
| 订单服务 | order_db | 5 |
| 支付服务 | payment_db | 1 |
| 物流服务 | shipment_db | 1 |
| 评论服务 | comment_db | 2 |
| 博客服务 | blog_db | 4 |
| 短信服务 | sms_db | 2 |
| 店铺服务 | store_db | 3 |
| **总计** | **9** | **29** |

### 2. 移除跨服务外键

**改造前：**
```sql
-- orders 表（在 shopping_platform_db 中）
FOREIGN KEY (user_id) REFERENCES users(id)
FOREIGN KEY (address_id) REFERENCES addresses(id)
```

**改造后：**
```sql
-- orders 表（在 order_db 中）
-- user_id BIGINT NOT NULL COMMENT '用户 ID（逻辑关联）'
-- 移除外键约束，通过应用层 API 调用获取用户信息
```

**改造前：**

## ⚠️ 注意事项

### 1. 数据一致性

由于移除了外键约束，需要在应用层保证数据一致性：

**解决方案：**
- 使用分布式事务（如 Saga 模式）
- 通过消息队列实现最终一致性
- 在代码层进行数据验证

### 2. 服务间调用

**示例场景：创建订单时需要验证用户和商品**

```java
// 订单服务代码示例
public Order createOrder(OrderDTO orderDTO) {
    // 1. 调用用户服务验证用户
    UserDTO user = userServiceClient.getUser(orderDTO.getUserId());
    if (user == null) {
        throw new IllegalArgumentException("用户不存在");
    }
    
    // 2. 调用商品服务验证商品和库存
    ProductDTO product = productServiceClient.getProduct(orderDTO.getProductId());
    if (product == null || product.getStock() < orderDTO.getQuantity()) {
        throw new IllegalArgumentException("商品库存不足");
    }
    
    // 3. 创建订单
    Order order = orderMapper.insert(orderDTO);
    
    // 4. 发送消息，异步扣减库存
    kafkaTemplate.send("inventory-topic", orderDTO);
    
    return order;
}
```

### 3. 查询优化

**跨服务查询场景：**

```java
// 方案 1：应用层组装
public OrderDetailVO getOrderDetail(Long orderId) {
    // 1. 查询订单
    Order order = orderMapper.selectById(orderId);
    
    // 2. 调用用户服务获取用户信息
    UserDTO user = userServiceClient.getUser(order.getUserId());
    
    // 3. 调用商品服务获取商品信息
    List<OrderItem> items = orderItemMapper.selectByOrderId(orderId);
    List<ProductDTO> products = productServiceClient.getProducts(
        items.stream().map(OrderItem::getProductId).collect(Collectors.toList())
    );
    
    // 4. 调用物流服务获取物流信息
    Shipment shipment = shipmentServiceClient.getShipmentByOrderId(orderId);
    
    // 5. 组装 VO
    OrderDetailVO vo = new OrderDetailVO();
    vo.setOrder(order);
    vo.setUser(user);
    vo.setProducts(products);
    vo.setShipment(shipment);
    
    return vo;
}
```

### 从旧数据库迁移到新数据库

如果已有旧数据库数据，需要进行迁移：

1. **导出数据**
```bash
mysqldump -u root -p shopping_platform_db > backup.sql
```

2. **修改 SQL 脚本**
- 将 USE 语句改为对应的数据库名
- 移除跨服务外键约束

python db_init.py  # 创建新数据库结构
# 手动导入数据到对应数据库
